Note: The response object is automatically passed as the first parameter in each defined hook, as well as the optional callback function (e.g., when using the $m.setPage() method). It contains valuable information that can be used for additional logic in your application. For example, the caller property is a string indicating the name of the API method or constructor that triggered the hook recieving the response object. If n/a is shown in the Response Data box above, indicates no change was detected/needed for the current page or view (i.e., no hooks triggered).
